가성비 바이브 코딩 가이드

Quality · Speed · Cost — 세 가지 균형을 잡는 것이 진짜 효율이다.

AI 코딩 도구는 빠르게 발전하고 있지만, 비싼 모델을 무분별하게 쓰는 것은 낭비다. 이 가이드는 Cursor, Opencode, vLLM을 전략적으로 조합하여 최소 비용으로 최대 퀄리티를 달성하는 방법을 다룬다.

이 가이드의 대상​

  • Cursor, Claude 등 AI 코딩 도구를 이미 사용해본 개발자
  • AI 비용 대비 생산성을 높이고 싶은 개발자
  • 반복 작업을 자동화하여 코딩 사이클을 단축하고 싶은 개발자

완전 입문자라면 기본적인 Cursor 사용법을 먼저 익히고 돌아오기를 권장한다.

핵심 원칙 3가지​

1. 역할에 맞는 모델을 쓴다​

모든 작업에 최고급 모델을 쓰는 건 스포츠카로 장을 보러 가는 것과 같다.

작업 유형예시적합한 모델
설계 · 판단아키텍처 결정, 복잡한 비즈니스 로직Claude Opus 4.6 등 고성능
코드 생성CRUD, API 엔드포인트, 테스트Gemini Flash, Kimi k2.5
반복 실행리팩터링, 문서화, 타입 힌트로컬 vLLM (무료)

2. Skills로 모델의 한계를 보완한다​

저렴한 모델도 **잘 설계된 Skills(프롬프트 템플릿)**가 있으면 고품질 출력이 가능하다. Skills = 프롬프트 템플릿 + 컨텍스트 압축 + 자동화 스크립트의 조합.

3. 도구를 역할 분리하여 병행 사용한다​

도구강점주요 역할
CursorIDE 통합, 실시간 수정설계 대화, 파일 편집
OpencodeCLI 자동화배치 처리, Subagent 실행
vLLM로컬 무료 실행반복 작업 오프로드

가이드 구성​

01. 전체 아키텍처     — 도메인 → Task → Skills 구조 설계
02. 모델 선정 전략    — 어떤 작업에 어떤 모델을 쓸까
03. Cursor 설정       — OpenRouter 연결 + Rules 등록
04. Opencode + vLLM   — 로컬 서버 + CLI 자동화 파이프라인
05. 실전 프롬프트     — 설계/자동화 단계별 프롬프트 예시
06. 실전 워크플로우   — 신규 기능 개발 A to Z
07. 체크리스트        — 비용 절감 점검 항목

빠른 시작 (TL;DR)​

아무것도 모르겠다면 아래 순서대로 시작하라.

  1. Cursor + OpenRouter 연결 → 03. Cursor 설정 참고
  2. .cursorrules에 Skills 등록 → 03. Cursor 설정 참고
  3. 도메인 설계는 Opus로, 반복 구현은 경량 모델로 → 02. 모델 선정 참고
  4. 반복 작업은 vLLM으로 오프로드 → 04. Opencode + vLLM 참고
